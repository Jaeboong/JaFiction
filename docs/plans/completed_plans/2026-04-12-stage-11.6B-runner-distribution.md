# Stage 11.6.B — Runner Distribution 구현 계획

> 작성일: 2026-04-12  
> 목표: 일반 사용자가 클릭 한 번으로 로컬 러너를 설치하고 자동 연결할 수 있도록 한다.

---

## 배경

- 웹 UI의 `ConnectConsentModal`에서 `no_claim` 시 3단계 설치 가이드 모달이 표시됨
- Step 3에서 `/api/runner/download?os=windows|mac|linux` 로 다운로드 요청
- 현재 해당 엔드포인트 없음, 바이너리 없음 → 404

---

## 구현 범위

### Phase 1 — 백엔드 다운로드 엔드포인트

**파일**: `packages/backend/src/routes/runnerDownload.ts`  
**등록**: `packages/backend/src/app.ts`

```
GET /api/runner/download?os=windows|mac|linux
  → 302 redirect to GitHub Releases URL
  → 파일명: jasojeon-runner-windows.exe / jasojeon-runner-mac / jasojeon-runner-linux

GitHub Releases URL 패턴:
  https://github.com/Jaeboong/jasojeon/releases/latest/download/jasojeon-runner-{os}
```

- `os` 파라미터 없거나 잘못된 경우 400
- 환경변수 `RUNNER_DOWNLOAD_BASE_URL` 로 URL 오버라이드 가능 (기본값: GitHub Releases)

---

### Phase 2 — Bun 바이너리 빌드

**파일**: `packages/runner/build.ts`  
**스크립트**: `scripts/build-runner-binaries.sh`

타겟:
| 파일명 | 타겟 |
|--------|------|
| `jasojeon-runner-windows.exe` | `bun-windows-x64` |
| `jasojeon-runner-mac-arm64` | `bun-darwin-arm64` |
| `jasojeon-runner-mac-x64` | `bun-darwin-x64` |
| `jasojeon-runner-linux` | `bun-linux-x64` |

엔트리포인트: `packages/runner/src/cli.ts` (신규)  
빌드 출력: `packages/runner/dist-bin/`

환경변수 번들:
- `JASOJEON_BACKEND_URL=https://xn--9l4b13i8j.com` 기본값으로 빌드 시 embed

---

### Phase 3 — CLI 엔트리포인트

**파일**: `packages/runner/src/cli.ts`

서브커맨드:
```
jasojeon-runner install    # 서비스 등록 + 즉시 시작
jasojeon-runner start      # 러너 실행 (서비스가 호출)
jasojeon-runner uninstall  # 서비스 제거
jasojeon-runner status     # 연결 상태 출력
```

`install` 흐름:
1. OS 감지
2. 서비스 등록 (아래 Phase 4)
3. 즉시 `start` 실행 → 러너 기동 → `/auth/device-claim` 호출

---

### Phase 4 — OS별 서비스 등록

#### Windows — Task Scheduler

```
schtasks /Create /SC ONLOGON /TN "JasojeonRunner" \
  /TR "<exe경로> start" /RL LIMITED /F
```
- 관리자 권한 불필요 (ONLOGON + LIMITED)
- 설치 직후 즉시 실행: `schtasks /Run /TN "JasojeonRunner"`

**파일**: `packages/runner/src/service/windows.ts`

#### macOS — LaunchAgents

```xml
~/Library/LaunchAgents/com.jasojeon.runner.plist
  ProgramArguments: [<exe경로>, "start"]
  RunAtLoad: true
  KeepAlive: true
```

```
launchctl bootstrap gui/$UID ~/Library/LaunchAgents/com.jasojeon.runner.plist
```

**파일**: `packages/runner/src/service/mac.ts`

#### Linux — systemd user service

```ini
~/.config/systemd/user/jasojeon-runner.service
  ExecStart=<exe경로> start
  Restart=on-failure
  WantedBy=default.target
```

```
systemctl --user daemon-reload
systemctl --user enable --now jasojeon-runner
```

**파일**: `packages/runner/src/service/linux.ts`

## 구현 순서 (우선순위)

1. **Phase 3** — `cli.ts` + `start` 서브커맨드 (기존 `index.ts` 재사용)
2. **Phase 4** — Windows 서비스 등록 (windows.ts 우선, 95% 사용자)
3. **Phase 2** — bun 빌드 설정
4. **Phase 1** — 백엔드 다운로드 엔드포인트
5. **Phase 5** — CI 릴리스 워크플로
